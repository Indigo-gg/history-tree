# PDF 知识库处理说明

本目录保存从 `D:\浏览器下载\（最全）初中历史知识点归纳汇总.pdf` 自动提取出的参考数据。

## 输出文件

- `source/history-pdf-full-text.txt`：PDF 全量文本提取结果。
- `source/history-pdf-extracted.md`：按册别和课切分后的 Markdown，方便人工阅读校对。
- `source/history-pdf-lessons.json`：按课切分的结构化数据，保留每课原文、关键词和候选节点。
- `source/history-pdf-manifest.json`：提取统计信息。
- `app/history-nodes.candidate.json`：适合导入 App 的候选知识节点草稿。

## 当前提取统计

- PDF 页数：55 页。
- 识别课级章节：98 个。
- 候选知识节点：588 个。

## 使用原则

这份 PDF 只能作为知识整理参考，不能直接等同于最终 App 知识库。

后续进入产品前需要做三层处理：

1. 校对史实：检查年份、人物、事件名称和教材版本是否与当前人教版及河南中考要求一致。
2. 改写解释：把原资料中的条目改成“背景-过程-结果-影响-易混点”的知识卡片，不直接搬运题目。
3. 建关系：补充时间先后、因果、对比、包含、影响等知识树关系。

## 推荐清洗流程

第一批建议先处理八年级上册：

- 鸦片战争
- 第二次鸦片战争
- 甲午中日战争
- 八国联军侵华
- 洋务运动
- 戊戌变法
- 辛亥革命
- 新文化运动
- 五四运动
- 中国共产党成立

处理顺序：

1. 从 `source/history-pdf-lessons.json` 读取对应课。
2. 人工确认每课的核心知识点。
3. 把候选节点合并为更适合学习的知识卡。
4. 给每个知识卡补充一句话解释和易混提醒。
5. 再把知识卡连接成专题知识树。

## 候选节点字段说明

`app/history-nodes.candidate.json` 中每个节点都有 `needsHumanReview: true`，表示它只是自动抽取的草稿。

关键字段：

- `title`：自动推断的节点标题，可能需要重命名。
- `sourceExcerpt`：来自 PDF 的参考段落。
- `keywords`：自动提取的关键词。
- `timeText`：可能包含时间信息。
- `people`、`background`、`process`、`result`、`influence`：自动分类的事实片段。
- `examPhrases`：可能适合中考表达的句子。
- `textbook`：册别和课题来源。

## 小测处理建议

资料里的填空或问答不要直接搬进 App。更合适的方式是：

- 从知识卡结构生成“理解检查”。
- 题目后重点展示解释和易混原因。
- 错题只用于标记薄弱知识点，不做复杂分数系统。
